# 커밋 메시지 작성 규칙

이 문서는 Recipick 백엔드 저장소의 커밋 메시지를 일관된 형식으로 작성하기 위한 규칙이다.
기존 커밋에는 소급 적용하지 않으며, 새로운 커밋부터 적용한다.

## 1. 기본 형식

커밋 메시지 제목은 다음 형식으로 작성한다.

```text
<type>(<scope>): <제목>
```

`scope`는 변경 영역을 명확하게 나타낼 필요가 있을 때만 작성한다.

```text
feat(member): 회원 가입 API 추가
fix(batch): 회원 배치 실행 예외 전파
docs: 커밋 메시지 규칙 문서 추가
```

## 2. 타입

타입은 변경의 주된 목적을 기준으로 다음 중 하나를 사용한다.

| 타입 | 사용 기준 |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 동작 변경 없는 코드 구조 개선 |
| `test` | 테스트 추가 또는 수정 |
| `docs` | 문서 추가 또는 수정 |
| `style` | 코드 포맷, 공백 등 동작에 영향을 주지 않는 변경 |
| `perf` | 성능 개선 |
| `build` | 빌드 설정 또는 의존성 변경 |
| `ci` | CI 설정 또는 스크립트 변경 |
| `chore` | 위 타입에 포함되지 않는 유지보수 작업 |
| `revert` | 이전 커밋 되돌리기 |

하나의 커밋에 여러 목적이 섞여 있다면 가장 중요한 타입을 고르는 대신 변경을 목적별 커밋으로 분리한다.

## 3. 범위

범위는 선택 사항이며 변경된 도메인이나 구성 요소를 소문자로 작성한다.

이 저장소에서는 다음과 같은 범위를 사용할 수 있다.

- `member`: 회원 도메인
- `post`: 게시글 도메인
- `batch`: 배치 작업
- `common`: 여러 도메인에서 공유하는 코드
- `config`: 애플리케이션 및 보안 설정
- `docs`: 프로젝트 문서

변경 영역이 여러 곳에 걸치거나 타입만으로 목적이 충분히 드러나는 경우에는 범위를 생략한다.

## 4. 제목

- 변경 내용을 한글로 간결하게 작성한다.
- 타입과 범위는 영문 소문자로 작성한다.
- 클래스명, 메서드명, API 경로 등 기술 식별자는 원문을 유지한다.
- 제목 끝에 마침표를 붙이지 않는다.
- 제목은 전체 커밋 제목을 기준으로 72자 이내로 작성한다.
- `수정`, `업데이트`, `작업`처럼 변경 내용을 알기 어려운 표현만 사용하지 않는다.

```text
# 권장
fix(member): Member UUID 생성 타입 오류 수정
refactor(common): ApiResponse 생성 로직 단순화

# 지양
[FEAT] Member Create
fix: 수정
update settings
```

## 5. 본문

제목만으로 변경 이유나 영향을 충분히 설명할 수 없을 때 본문을 작성한다.
제목과 본문 사이에는 빈 줄을 하나 두고, 본문에는 무엇을 변경했는지보다 변경한 이유와 영향을 중심으로 적는다.

```text
fix(batch): 회원 배치 실패 응답 처리

배치 실행 예외가 로그에만 남아 API가 성공으로 응답하던 문제를 해결한다.
실행 실패를 호출자에게 전파해 공통 예외 처리기가 실패 응답을 생성하도록 한다.
```

## 6. 이슈 연결

관련 이슈는 본문 다음의 footer에 작성한다.

```text
Refs: #123
Closes: #456
```

- `Refs`는 관련 이슈를 참조할 때 사용한다.
- `Closes`는 해당 커밋으로 이슈가 해결될 때 사용한다.

## 7. 호환성을 깨는 변경

기존 API나 데이터 형식과 호환되지 않는 변경은 타입 또는 범위 뒤에 `!`를 붙인다.
본문 아래에는 `BREAKING CHANGE:`로 구체적인 영향을 작성한다.

```text
feat(api)!: 회원 응답 구조 변경

BREAKING CHANGE: 숫자 회원 ID를 memberUuid로 대체
```

## 8. 전체 예시

```text
feat(member): 회원 가입 API 추가
```

```text
fix(batch): 회원 배치 실행 예외 전파
```

```text
refactor(common): 공통 응답 생성 로직 단순화
```

```text
test(post): 게시글 저장 서비스 테스트 추가
```

```text
docs: 커밋 메시지 규칙 문서 추가
```

```text
feat(api)!: 회원 응답 구조 변경

회원 식별자 노출 방식을 UUID 기반으로 통일한다.

BREAKING CHANGE: 숫자 회원 ID를 memberUuid로 대체
Refs: #123
```
